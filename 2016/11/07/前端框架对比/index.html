前端框架对比

前端CSS框架：

Bootstrap VS Foundation VS Materialize

Bootstrap

  • Github 关注度
    Start: 100,212
  • 中文文档支持
    中文文档齐全，国内用户量最多
  • 兼容性
    Bootstrap 2.2 支持 IE7.0+ Bootstrap 3 支持 IE8.0+
  • 优点
    用户群体大，文档齐全，兼容性好。
  • 缺点
    样式固定，效果也比较普通，不过样式方面可以以它为基础，来做自己的定制。

    Foundation

  • Github 关注度
    Start: 24,025
  • 中文文档支持
    中文文档欠缺，用户量较少
  • 兼容性
    Foundation 支持 IE9.0+
  • 优点
    REMs来定义元素宽度，更好的兼容移动端，网格灵活，移动端优先。
  • 缺点
    中文文档不全，社区也不活跃，样式比较普通。

    Materialize

  • Github 关注度
    Start: 21,112
  • 中文文档支持
    有中文文档，国内开始有人关注并且使用。
  • 兼容性
    Chrome 35+,
    Firefox 31+,
    Safari 7+,
    IE 10+
  • 优点
    样式设计感强，效果不错，组件丰富，拥有中文文档。
  • 缺点
    目前使用的人不算很多，正在逐步提升，酷炫的效果同样意味着在旧版的浏览器下的表现一般，类名比较多，容易重复。

    总结：

    前端css框架我选了这三个进行对比，这三个的主要作用都是可以快速搭建出一个还看得过去的网站，里面也有很多比较实用的小组件，例如模态框／下拉框／轮播图／等等，其实基本上大同小异，主要是设计风格的不同。三个框架都支持响应式布局，兼容性基本都还不错。bootstrap已经很好用了，快速，简洁，由于其受欢迎程度好，所以使用的时候基本上不会遇到什么困难，兼容性也做的很好。fundation的话，国内用的人比较少，我也没太深入的了解，因为其作用跟风格和bootstrap差不多，并且还没有中文文档支持，所以国内关注的人很少。Materialize是基于material design 设计出来的一套CSS框架，material design是由谷歌提出来的一种设计风格，其特点就是干净、简约、其动画效果更加突出。就这套框架而言，其实也都继承了bootstrap的特点点，基本上bootstrap有的东西，他也都有，但是效果就更加酷了，但是有个缺点就是 样式类太烦杂，还有文件稍稍比bootstrap大那么一丢丢，兼容性还是差了一点，所以，用来做自己的小项目玩玩是不错的。

前端M*V框架：

Angular

  • Github 关注度
    Start: 51,927
    相关项目：156,261
  • 中文文档支持
    由于angular兴起得最早，所以中文文档比较齐全，教程也多。
  • 技术背景
    由谷歌团队开发，技术背景雄厚，但他们并没有把angular应用在他们核心的产品上。
  • 上手难度
    由于发展较早，angular学起来会有很多教程，上手应该不会很难，不过很多以前用angular的都选择了vue，因为写起来方便？？上手难度3.5/5
  • 适用领域
    Angular 使用双向绑定即：界面的操作能实时反映到数据，数据的变更能实时展现到界面。功能大而全，适用于一些需要实时反映交互／数据的复杂场景。
  • 视图渲染方式
    AngularJS的工作原理是:HTML模板将会被浏览器解析到DOM中, DOM结构成为AngularJS编译器的输入。AngularJS将会遍历DOM模板, 来生成相应的NG指令,所有的指令都负责针对view(即HTML中的ng-model)来设置数据绑定。因此, NG框架是在DOM加载完成之后, 才开始起作用的。
  • 优点
    发展较早，模板功能强大而且丰富，模块化比较大胆的引入了Java的一些东西（依赖注入）？能够很容易的写出可复用的代码，代码生命周期长。
  • 缺点
    太笨重了，没有让用户选择一个轻量级的版本；性能上不如其他两个框架，而且写起来比较繁琐。 高级版本跟低版本API改动较大，很难兼容。

React

  • Github 关注度
    Start: 48,641
    相关项目：110,135
  • 中文文档支持
    react 有中文文档，但是新版本API更改比较多，要了解最新的改动，有时还是得去看英文文档
  • 技术背景
    react 由Facebook团队开发，并且应用在他们的产品之上。
  • 上手难度
    react出名全家桶，因为要使用 react并不止只学react那么简单，而且还有JSX、react-router、redux、ES6、webpack、virtual dom等等，如果要拓展到移动端还有react-native，所以上手难度4.5/5。
  • 适用领域
    React为单向数据流模式给定原始界面（或数据），施加一个变化，就能推导出另外一个状态（界面或者数据的更新），适合一些需要交互比较频繁的场景，其组件化思想也适合一些小团队的协作开发。
  • 视图渲染方式
    React 的渲染建立在 Virtual DOM 上——一种在内存中描述 DOM 树状态的数据结构。当状态发生变化时，React 重新渲染 Virtual DOM，比较计算之后给真实 DOM 打补丁。Virtual DOM 提供了函数式的方法描述视图，它不使用数据观察机制，每次更新都会重新渲染整个应用，因此从定义上保证了视图与数据的同步。它也开辟了 JavaScript 同构应用的可能性。
  • 优点
    React很大的特点就是“轻”，Virtual DOM理念先进，性能好，组件化思想开发起来比较快，还有react-native可以延展到移动端，社区活跃，是近两年来最火的前端框架，没有之一。
  • 缺点
    学习曲线比较陡峭，要很好的使用react必须要用它的那一整套东西。而且不同版本之间的API改动比较大，要用最新的东西需要参考英文文档。

Vue

  • Github 关注度
    Start: 26,470
    相关项目：8,060
  • 中文文档支持
    vue 本身作者就是国人，中文社区比较齐全。
  • 技术背景
    vue 由国人尤雨溪开发，最新推出的weex框架是用vue将vue应用到移动端，由vue作者跟淘宝团队参与开发。
  • 上手难度
    Vue轻量级框架，组件化应用思想，而且代码结构清晰，UI、样式、逻辑分离。上手难度2.5/5
  • 适用领域
    Vue同样可以实现双向数据流， 比起angular有更好的性能，并且非常非常容易优化。适用于一些轻量级的业务场景。
  • 视图渲染方式
    Vue.js 不使用 Virtual DOM 而是使用真实 DOM 作为模板，数据绑定到真实节点。Vue.js 的应用环境必须提供 DOM。Vue.js 有时性能会比 React 好**，而且几乎不用手工优化。
  • 优点
    同样是个轻量级框架，vue使用真实dom，代码结构清晰，写起来舒服，文档齐全，容易上手，借口灵活，性能好。
  • 缺点
    目前关注度还不是特别高，社区也不如react那边火热，不过好在作者是国人，可以直接在中文社区提问，更多的缺点我查不到了，主要是比较新，还不是特别多人关注，不过2016年以来关注的人越来越多，而且vue2.0已经上线，所以性能上会有很大提升。

总结：

三个框架而言，如果是要开展新的项目的话，angular很多人都不推荐用了，主要是react和vue的对比，目前而言react的受关注度还是比vue高的，主要原因我觉得是react-native的驱动，毕竟用JS可以写移动端APP这个特点还是非常吸引人的，但是就我对react的理解而言，如果已经学会了它的那套全家桶的话开发一些复杂场景的话，还是很好的，性能也高，而且其组件化思想，多人协作的时候可以很好合作。vue的话2016年才开始火起来，比react晚了一年，但其势头有赶超react的样子，并且相对于react-native，它也准备有一套Weex框架开始涉及移动端开发，很有前途，再加上它写起来真的比较简单，中文文档齐全，学起来应该不会比react难。不过，这也只是说开发webapp这类交互比较复杂的场景，如果只是简单的展示页的话，功能没那么复杂，用jQuery已经够了，插件齐全，写起来简单，容易理解。不过以学习为目的的话，我觉得我下一步应该就回去入手vue了。
以上总结纯粹个人的一些愚见，如有错漏，恳请指出。
